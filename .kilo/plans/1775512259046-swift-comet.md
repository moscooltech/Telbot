# Plan: Fix Audio Script Generation

## Problem
1. The AI generates audio scripts with "scene 1", "scene 2", "step 1" references instead of pure narration content
2. The narration is not being properly handled from extraction (AI response) to transcription (gTTS) to burn (video)

## Pipeline Analysis

### Stage 1: Extraction (AI Response → Narration Text)
**Location:** `services/scene_generator.py:91-92`
```python
narrations = [s.get("spoken_script", "") for s in raw_scenes]
```
- Extracts `spoken_script` from each scene in JSON response
- **Potential issue:** If AI returns different field name or malformed JSON, extraction could fail

### Stage 2: Transcription (Text → gTTS Audio)
**Location:** `bot/handlers.py:123-134` + `services/audio_processor.py:18-38`
```python
for i, narration_text in enumerate(unique_narrations):
    path, dur = ap.generate_single_narration(narration_text, i)
```
- Passes each narration text to gTTS
- Creates file `scene_{index:03d}.mp3`
- **Potential issue:** 
  - Empty or None text being passed
  - Index not incrementing properly (same file overwritten)
  - gTTS failing silently

### Stage 3: Merge Audio
**Location:** `services/audio_processor.py:40-66`
- Merges all narration files into single `final_audio.mp3`
- **Potential issue:** Files not in correct order, wrong files included

### Stage 4: Burn to Video
**Location:** `services/video_processor.py:67-96`
- Adds merged audio to final video
- **Potential issue:** Audio not properly aligned with video duration

## Root Causes to Address

1. **Scene/Step references in narration** - System prompt fix (done, needs testing)
2. **Fallback generates fake content** - Replace with error handling
3. **Potential extraction issues** - Add validation to ensure narrations are valid
4. **Track which narration goes to which scene** - Add clarity in logging

## Proposed Fixes

### Fix 1: Remove hardcoded fallback, add proper error handling
- In `scene_generator.py`, if AI fails after retries, raise exception instead of returning fake content

### Fix 2: Add narration validation
- Before generating audio, validate that:
  - Narrations are not empty
  - Narrations don't contain "scene X", "step X" patterns
  - Each narration is different from others
- If validation fails, either retry or raise error

### Fix 3: Ensure index uniqueness in audio generation
- Verify that `enumerate(unique_narrations)` is using correct index
- Each scene should get unique audio file with unique index

## Files to Modify
1. `services/scene_generator.py` - Replace fallback with error, add validation
2. `bot/handlers.py` - Add narration validation before audio generation
